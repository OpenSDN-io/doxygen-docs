13 // Tag helps in attaching well defined attribute to any object
14 // in system. Currently tag attached to VMI, VM, VN and Project are of
15 // significance
16 //
17 // Agent builds tag list directly at VMI.
18 // Tag built at VMI includes all the tags that are defined at VMI, VM,
19 // VN and project.
20 //
21 // Derivation of tag at VMI:
22 // VMI config processing goes thru below links to build tag list
23 // VMI --> Tag
24 // VMI --> VM --> Tag
25 // VMI --> VN --> Tag
26 // VMI --> Project --> Tag
27 //
28 //Label can be multiple at VMI.
29 // VMI will have only one each tag of type application, site,
30 // deployment and tier, and priority would be in above order of link
31 // Tag list built at VMI would be exported to control-node.
32 // Tag list would be used in ACL lookups
