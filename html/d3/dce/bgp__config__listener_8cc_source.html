24 // Populate the policy with entries for each interesting identifier type.
25 //
26 // Note that identifier types and metadata types will have the same name
27 // when there's a link with attributes. This happens because we represent
28 // such links with a "middle node" which stores all the attributes and add
29 // plain links between the original nodes and the middle node. An example
30 // is "bgp-peering".
31 //
32 // Additional unit tests should be added to bgp_config_listener_test.cc as
33 // and when this policy is modified.
34 //
